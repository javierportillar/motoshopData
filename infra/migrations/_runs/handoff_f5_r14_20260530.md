## ⚠️ Notas importantes

1. **MySQL 5.0**: las migrations usan `SET PASSWORD FOR` (compatible 5.0), `TEXT` en vez de `JSON`, `TIMESTAMP` en vez de `DATETIME DEFAULT CURRENT_TIMESTAMP`, `utf8` en vez de `utf8mb4`.
2. **Idempotency**: las migrations tienen `UNIQUE KEY uq_idempotency (idempotency_key)`. Si se viola, el repo captura `IntegrityError` y devuelve el registro existente (HTTP 200).
3. **Rollback**: si algo falla, ejecutar `DROP TABLE IF EXISTS app_alert_actions, app_audit_log;` y `DROP USER 'app_writer'@'%';` para limpiar.
4. **R14 no destructivo**: Prophet/LightGBM quedan en `docs/archive/` y las tablas `gold.forecast_*_sku` en Databricks siguen existiendo (solo no se regeneran).
