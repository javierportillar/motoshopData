# F6-001 · Particionado app_audit_log — No aplicable

**Fecha:** 2026-05-30 12:20
**Ejecutó:** Runtime Dev (Windows)
**MySQL version:** 5.0.45-community-nt

## Resultado

**NO APLICABLE.** MySQL 5.0 no soporta particionado de tablas.
- information_schema.PARTITIONS no existe en MySQL 5.0.
- La migration F6-001-app_audit_log_partition.sql requiere RANGE COLUMNS y PARTITION BY, introducidos en MySQL 5.1+.
- No se ejecutó la migration para no romper la tabla existente.

## Evidencia

`
mysql> SELECT VERSION();
+-------------------------+
| 5.0.45-community-nt     |
+-------------------------+

mysql> SELECT PARTITION_NAME FROM information_schema.PARTITIONS WHERE TABLE_NAME='app_audit_log';
ERROR 1109 (42S02): Unknown table 'partitions' in information_schema
`

## Decisión

Particionado diferido a F7, que requiere migración de MySQL 5.0 a 5.7+ o 8.0.
Mientras tanto, pp_audit_log se mantiene como tabla única. El volumen actual
(~2 filas) no justifica partitioning — es preventivo para escala futura.
